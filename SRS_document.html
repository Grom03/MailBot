Введение


Целью этого документа является описание требований к программному обеспечению в виде телеграмм бота, который присылает уведомления из почты по выбранным критериям.


Область


Телеграмм бот должен пересылать письма из почты в телеграмм по выбранным критериям, таким как имя отправителя, тема письма и тд.


Определения


Бот в Telegram – это разновидность чат-бота, который позволяет пользователю осуществлять определенные действия через мессенджер.

API – описание способов взаимодействия одной компьютерной программы с другими.

<Почтовый клиент - программа для чтения электронной почты или, более формально, message user agent (MUA) или mail user agent - это компьютерная программа, используемая для доступа к электронной почте пользователя и управления ею.



Обзор


Данный документ содержит описание продукта и его технические требования.

Описание


Конечный продукт будет представлять из себя Telegram-бота. Все вычисления будут производиться на удаленном сервере. Серверная часть будет реализована на языке python с использованием библиотек для работы с СУБД. В базе данных будет храниться необходимая информация о пользователях.
Пользователь продукта – это человек с существующими Telegram-аккаунтом и электронной почтой, который хочет получить возможность взаимодействовать со своим электронным почтовым ящиком, не покидая Telegram.
Продукт будет доступен на всех платформах, под которые существуют клиенты Telegram с поддержкой ботов. (Microsoft Windows, GNU/Linux, MacOS, Android, IOS, etc.)

Продукт не будет зависеть от наличия каких-либо сторонних библиотек или софта, кроме самого клиента Telegram.


Особые требования

Функциональные требования


Телеграмм бот со следующими функциями:

  1. Возможность регистрации через API в почте при помощи логина и пароля пользователя

  2. Выбор критериев для пересылки писем, таких как:

    1. Отправитель письма

    2. Тема письма

  3. Выбор формата вывода письма в телеграмм.

    1. Тема и отправитель письма

    2. Полный текст письма

    3. Полный текст письма со всеми вложениями

  4. Вывод недавних писем от конкретного отправителя

Удобство использования


Бот должен быть интуитивно понятен, для каждой функции должны быть перечислены все возможные варианты действий.


Надежность


Бот должен работать быстро (от 3 до 7 секунд в зависимости от количества писем) и корректно, при возникновении ошибок присылать их текстовым сообщением с расшифровкой ошибки.


Производительность


Время ответа на запрос пользователя не должно превышать 10 секунд.


Поддержка


Будут использоваться система контроля версий, unit-тестирование.


Ограничения проектирования


Бот должен быть написан при помощи Python с использованием SQL.

Интерфейсы


Интерфейс пользователя


Телеграмм бот с выбором необходимых функций из списка и вводом текстовых значений в виде сообщений.


Интерфейс с аппаратным обеспечением


Бот запускается с любого программного обеспечения при помощи приложения Telegram.


Интерфейс с программным обеспечением


Планируется взаимодействие с API Telegram, а также с API почт.
Используемые библиотеки - logging, imaplib, email, telegram.
